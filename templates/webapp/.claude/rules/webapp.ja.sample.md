# webapp.md の使い方(人間向け解説)

Webアプリ(前後端分離)で Claude が起こしがちな事故を防ぐルールです。

## 典型的な事故と対応ルール

| よくある事故 | 防ぐルール |
|-------------|-----------|
| フロントから DB 直叩きの「動くコード」を書く | API 経由の強制 |
| API を変えてフロントの修正を忘れる | 境界の同時更新 |
| 秘密鍵をフロントの env に書く | 環境変数の分離 |

## カスタマイズ例

```markdown
- フロントは Next.js App Router のみ。Pages Router のコードを書かない
- API パスは /api/v1/ プレフィックス必須
- CORS 設定を変更する場合は必ず確認を取る
```

## さらに効果を上げるには

frontend/ と backend/ それぞれに CLAUDE.md を置くと、
Claude はそのディレクトリで作業するときだけ固有ルールを読みます
(CLAUDE.md の三層階層)。
