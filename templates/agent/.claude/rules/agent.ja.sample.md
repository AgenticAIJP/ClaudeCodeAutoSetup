# agent.md の使い方(人間向け解説)

MCP・自動化エージェント開発向けのルールです。

## なぜ「1エージェント = 1責務」か

責務が混ざったエージェントは、プロンプトが肥大化して
動作が不安定になり、デバッグも困難になります。
小さいエージェントを組み合わせる方が、テストも改善も楽です。

## Before / After

✅ ルールあり → ツールの入出力に型があり、失敗時の挙動が予測できる
❌ ルールなし → API キーがプロンプトに直書きされ、GitHub に流出する事故

## カスタマイズ例

```markdown
- LLM 呼び出しは必ず client.ts のラッパー経由(コスト計測のため)
- エージェント間の通信は JSON のみ。自然言語での受け渡し禁止
- 実行ログは logs/ に構造化形式(JSONL)で残す
```
