# api.md の使い方(人間向け解説)

API サービスで Claude が起こしがちな事故を防ぐルールです。

## 最重要ポイント: API_SPEC.md の同時更新

Claude はエンドポイントを追加するのは得意ですが、
**仕様書の更新は頼まないと忘れます**。
「同じ変更で更新する」とルール化することで、仕様書が常に最新に保たれ、
次のセッションの Claude が正確な仕様を読めるようになります(好循環)。

## Before / After

✅ ルールあり → routes は10行程度、ロジックは services、仕様書も自動更新
❌ ルールなし → route ハンドラに 200 行のロジック、仕様書は初日のまま

## カスタマイズ例

```markdown
- 認証は必ず middleware/auth.ts を通す。個別実装禁止
- ページネーションは cursor 方式で統一
- レートリミットの変更は必ず確認を取る
```
