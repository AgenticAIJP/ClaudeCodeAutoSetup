# Documentation Site Rules

- Create articles from the boilerplate in `templates/` (front matter included)
- Sample code never lives inline in articles; put it in `examples/`, verify it runs, then quote it
- Never skip heading levels (h3 follows h2)
- One article = one topic. Split when it grows long
- State the target reader level (beginner/intermediate) in the article's front matter
- No broken links: when renaming a file, update every referrer in the same change
