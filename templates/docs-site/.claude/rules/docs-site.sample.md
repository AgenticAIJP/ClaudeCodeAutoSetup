# docs-site.md の使い方(人間向け解説)

学習サイト・技術ブログ運営向けのルールです。

## なぜ examples/ に分離するか

記事内のコードは「書いた時点では動く」が、時間とともに腐ります。
`examples/` に動くコードとして置けば、CI でテストでき、
「実行したらエラーになるチュートリアル」を防げます。

## Before / After

✅ ルールあり → 全記事が同じ構成・同じトーン。サンプルは常に動く
❌ ルールなし → 記事ごとに構成バラバラ、コピペしたら動かないコード

## カスタマイズ例

```markdown
- 文体は「です・ます」調で統一
- 専門用語の初出時は必ず1行の説明を付ける
- スクリーンショットは images/ に置き、代替テキスト必須
```
