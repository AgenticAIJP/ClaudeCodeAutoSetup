# Coding Style Rules

- Write code, comments, and identifiers in English
- One function stays under 50 lines; split it if it grows beyond that
- No magic numbers. Use named constants
- Never swallow errors. Log them or propagate them upward
- Before fixing a bug, write a reproducing test first (prevents regressions)
- Commit messages follow Conventional Commits (feat: / fix: / docs: ...)
